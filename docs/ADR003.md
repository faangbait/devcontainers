# ADR003

A lineage is a sequence of related build layers. Within each lineage, inputs
are ordered from most mature to least mature.

Maturity means how stable and repeatable an input is expected to be, not how
good or old the software is. A fixed release from an established source is
more mature than a floating package version; a mutable branch or remote
installer is less mature still. This is a relative ordering, not a promise
that any input will never change.

Putting mature inputs first lets their layers remain cached when younger
inputs change. Putting mutable inputs last limits routine cache invalidation
and makes the least reproducible acquisitions easy to identify.

When adding an input, place it after inputs that are less likely to change and
before inputs that are more likely to change. If two inputs are similarly
stable, preserve the existing order.
